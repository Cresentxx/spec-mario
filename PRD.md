# Product Requirements Document — 超级马里奥 (Super Mario Bros.)

> 版本: 1.0 | 日期: 2026-05-19 | 状态: Draft

---

## 1. 概述

### 1.1 产品愿景

复刻经典任天堂红白机 (NES) 游戏《Super Mario Bros.》(1985) 的核心玩法体验，以 Python + Pygame 开发一个功能完整的超级马里奥桌面游戏。玩家操控马里奥穿越 2 个世界、8 个关卡，体验原版全部基础机制。最终打包为 Windows .exe 可执行文件独立运行。

### 1.2 目标平台

- Windows 10 / 11 PC 客户端
- 打包格式: 单一 .exe 可执行文件 (无需安装 Python 环境)
- 分辨率: 原生 256×240 像素逻辑分辨率，等比缩放适配屏幕

### 1.3 技术栈

- 游戏引擎: Pygame
- 语言: Python 3.10+
- 物理: 自研轻量物理引擎 (AABB 碰撞检测)
- 音频: Pygame.mixer (基于 SDL2_mixer)
- 打包工具: PyInstaller / Nuitka (输出 .exe)

## 2. 角色与状态

### 2.1 玩家角色

**Mario** — 主角，红色帽子工装裤

### 2.2 玩家状态机

```
Small Mario (小) ←→ Super Mario (大) ←→ Fire Mario (火球)
        ↑ 死亡 (碰到敌人/掉坑)          ↑ 死亡
        ↓ 蘑菇                          ↓ 蘑菇 → 变为 Super
        ↓ Fire Flower                    ↓ Fire Flower
```


| 状态  | 外观变化         | 能力                       | 受击结果   |
| ----- | ---------------- | -------------------------- | ---------- |
| Small | 小体型，1 格高   | 基础移动/跳跃              | 死亡       |
| Super | 大体型，2 格高   | 可顶碎砖块                 | 退回 Small |
| Fire  | 大体型，白色衣服 | 可发射火球 (最多 2 发同屏) | 退回 Super |

### 2.3 星星无敌

拾取 Starman 后进入无敌状态，持续约 10 秒。无敌期间:

- 接触任何敌人直接击杀
- 角色闪烁发光
- 免疫岩浆和陷阱伤害 (不免疫掉坑)

## 3. 核心操控

### 3.1 键盘映射


| 操作        | 按键            |
| ----------- | --------------- |
| 移动左      | ← / A          |
| 移动右      | → / D          |
| 跳跃 (确认) | Z / Space / ↑  |
| 加速跑      | Shift (按住)    |
| 发射火球    | Z (Fire 状态)   |
| 暂停        | Enter / P       |

> Pygame 通过 `pygame.key` 模块处理键盘输入，使用 `KEYDOWN` / `KEYUP` 事件驱动模式。

### 3.2 移动物理参数


| 参数           | 值 (像素/帧, 60fps) |
| -------------- | ------------------- |
| 步行最大速度   | 1.5 px/frame        |
| 跑步最大速度   | 2.5 px/frame        |
| 加速度         | 0.06 px/frame²     |
| 减速度 (松键)  | 0.10 px/frame²     |
| 跳跃初速度     | -4.0 px/frame       |
| 重力加速度     | 0.25 px/frame²     |
| 最大下落速度   | 4.5 px/frame        |
| 加速跳最高高度 | 约 4.5 格           |
| 普通跳最高高度 | 约 4 格             |

### 3.3 操控特性

- **跳跃手感**: 按住跳跃键时间越长跳得越高 (可变跳跃高度)
- **空中控制**: 跳跃后仍可在空中调整水平方向
- **加速跑**: 按住加速键同时移动可达到更高速度，高速下跳跃距离更远
- **刹车**: 高速运行中反方向按键产生刹车减速
- **转身**: 短按反方向原地转身

| 敌人                       | 行为                   | 弱点        | 分值 |

- 飞行龟壳可伤害玩家

## 5. 道具系统

### 5.1 可拾取道具


| 道具                        | 来源      | 效果                                     |
| --------------------------- | --------- | ---------------------------------------- |
| **Super Mushroom** (红蘑菇) | 问号方块  | Small → Super，若已是 Super 则 +1000 分 |
| **Fire Flower** (火球花)    | 问号方块  | 任意状态 → Fire，Small 时先变大         |
| **Starman** (无敌星)        | 问号方块  | 10 秒无敌，持续闪烁                      |

| **Coin** (金币)             | 散布/方块 | 每 100 枚 = +1 生命                      |

### 5.2 道具出现规则

- 蘑菇从方块中弹出后向右移动，遇壁反弹
- 火球花直接在方块上方出现
- 无敌星从方块弹出后向右弹跳移动
- 绿蘑菇在特定隐藏方块中出现，行为同红蘑菇

## 6. 方块系统

### 6.1 方块类型


| 方块                       | 行为                                        |
| -------------------------- | ------------------------------------------- |
| **Question Block (?)**     | 从下方顶击后掉落道具，变为空方块            |
| **Brick Block** (砖块)     | Super/Fire 状态下顶击碎裂；Small 状态仅弹跳 |
| **Hidden Block**           | 不可见，顶击后出现，通常含绿蘑菇或金币      |
| **Used Block**             | 已被激活的方块，实心，不可再互动            |
| **Hard Block** (地面/墙壁) | 不可破坏的实心方块                          |

### 6.2 方块互动规则

- 从下方顶击 (Player 头部碰到方块底面)
- 砖块碎裂动画: 4 个碎片向四周飞散
- 方块弹跳动画: 被顶时产生短暂上弹效果
- 道具从方块中弹出的动画

## 7. 关卡设计

### 7.1 世界结构

```
World 1: 地面主题 (Overworld)  — 入门难度，标准地面关卡
World 2: 城堡主题 (Castle)     — 火焰陷阱，高难度，Boss 关卡
```

### 7.2 每世界关卡


| 关卡   | 类型                     |
| ------ | ------------------------ |
| X-1    | 标准关卡                 |
| X-2    | 进阶关卡 (难度提升)      |
| X-3    | 挑战关卡 (高难度)        |
| X-4    | Boss 关卡 (World 2 为城堡 Boss) |

共计 **8 个关卡** (2 World × 4 Level)。

### 7.3 关卡主题与视觉


| 主题             | 地面色 | 背景色 | 方块色        | 特殊             |
| ---------------- | ------ | ------ | ------------- | ---------------- |
| Overworld (地面) | 橙砖色 | 天蓝   | 橙色问号/砖块 | 树木、灌木、云朵 |
| Castle (城堡)    | 黑色   | 黑色   | 灰色方块      | 火焰，岩浆       |

### 7.4 关卡元素


| 元素                           | 说明                                   |
| ------------------------------ | -------------------------------------- |
| **Pipe** (管道)                | 可进入的绿色管道，部分通向隐藏金币房间 |
| **Flagpole** (旗杆)            | 关卡终点，下滑得分 (越高分越多)        |
| **Stairs** (阶梯)              | 方块堆叠的阶梯结构，通往旗杆           |
| **Gap** (悬崖)                 | 掉落即死                               |
| **Moving Platform** (移动平台) | 水平或垂直移动的平台                   |
| **Fire Bar** (火焰棒)          | 城堡中旋转的火焰棒 (World 2)           |
| **Lava** (岩浆)                | 掉落即死 (World 2 城堡关卡)            |

### 7.5 隐藏区域

- **Coin Rooms**: 进入特定管道后的隐藏金币房间
- **Hidden Blocks**: 需要顶击才能出现的隐藏方块

## 8. 计分与生命系统

### 8.1 计分规则


| 动作                  | 分值                                       |
| --------------------- | ------------------------------------------ |
| 踩踏敌人              | 100                                        |
| 顶击敌人 (砖块碎裂)   | 100                                        |
| 踢壳连杀              | 200→400→800→1000→2000→4000→8000→1UP |
| 火球击杀              | 200                                        |
| 拾取金币              | 200                                        |
| 顶金币方块            | 200                                        |
| 拾取蘑菇              | 1000                                       |
| 拾取火球花            | 1000                                       |
| 拾取无敌星            | 1000                                       |
| 旗杆 (最高点)         | 5000                                       |
| 旗杆 (中高)           | 2000~4000                                  |
| 旗杆 (中低)           | 100~800                                    |
| 剩余时间 → 分数      | 每单位 50 分                               |
| 烟花 (旗杆尾数 1/3/6) | 每发 500 分                                |

### 8.2 生命系统

- 初始生命: **3 条**
- 获得额外生命:
  - 拾取 1-UP 蘑菇
  - 收集 100 枚金币
  - 连续踢壳击杀第 8 个敌人
  - 达到特定高分
- 死亡条件:
  - 碰到敌人 (Small 状态)
  - 掉入悬崖/岩浆
  - 时间耗尽
  - 被挤压 (墙壁与平台之间)

### 8.3 Game Over 与 Continue

- 生命耗尽 → Game Over 画面
- Continue: 可从当前 World 的第一关重新开始 (非 X-1，而是 World 起点)
- 最高分记录: 本地存储 Top 5 排行榜

## 9. 摄像机与画面

### 9.1 摄像机规则

- **水平滚动**: 玩家向右移动时摄像机跟随，**不可回退** (原版核心规则)

- **垂直滚动**: 关卡高度超出屏幕时垂直跟随
- 死亡后回到检查点重新开始

## 11. 游戏流程

### 11.1 游戏状态流转

```
Title Screen
    ↓ (Start)
World Map (显示当前 World-X-Y)
    ↓
Level Gameplay
    ↓ (通关)
Score Tally → Next Level
    ↓ (死亡)
Life -1 → Restart Level
    ↓ (Life = 0)
Game Over → Title Screen / Continue
```

### 11.2 通关流程

1. 玩家接触旗杆底部
2. 自动下滑旗杆动画 (根据高度得分)
3. 走入城堡建筑
4. 如满足条件播放烟花 (时间尾数为 1, 3, 6)
5. 显示关卡计分: 剩余时间 × 50
6. 进入下一关

### 11.3 城堡通关 (World 2-4)

1. 击败 Bowser (踩断桥梁或火球击杀)
2. 显示 "World 2 Clear" 信息
3. 播放结局动画
4. 游戏通关

### 11.4 Second Quest

通关后可选择重新开始:

- 所有 Goomba 替换为 Buzzy Beetle
- 敌人移动速度提升
- 火球花全部变为 Super Mushroom
- 新增隐藏要素

## 13. 技术要求

### 13.1 渲染

- 逻辑分辨率: 256 × 240 像素 (NES 原始分辨率)
- 窗口大小: 默认 768 × 720 (3× 缩放)，支持全屏模式
- 等比缩放: 使用 `pygame.transform.scale` 将逻辑画面缩放至窗口大小
- 像素完美: `pygame.SCALED` 标志 + 禁用平滑插值，保持像素风格
- 目标帧率: 60 FPS，通过 `pygame.time.Clock.tick(60)` 控制
- 精灵图 (Sprite): 使用 `pygame.sprite.Sprite` + Sprite Sheet 切片加载

### 13.2 碰撞检测

- 采用 AABB (轴对齐包围盒) 碰撞检测，利用 `pygame.Rect` 和 `colliderect()`
- 分轴检测: 先检测 X 轴碰撞再检测 Y 轴碰撞
- 碰撞层: 地形层、敌人层、道具层、玩家层
- 可选用 `pygame.sprite.groupcollide()` / `spritecollide()` 简化碰撞分组

### 13.3 关卡数据

- 使用 Tile Map (瓦片地图) 格式存储关卡数据
- 每个 Tile 大小: 16 × 16 像素
- 关卡尺寸: 16 × 15 tiles (屏幕大小)，横向可扩展至 200+ tiles
- 数据格式: JSON 文件，包含地形数据、敌人配置、道具配置
- 关卡文件通过 `pygame.image.load()` 加载瓦片精灵图

### 13.4 状态管理

```python
@dataclass
class GameState:
    current_world: int          # 1-2
    current_level: int          # 1-4
    player_state: str           # 'small' | 'super' | 'fire'
    score: int
    coins: int
    lives: int
    time_remaining: int
    camera_x: float
    entities: list[Entity]
    tile_map: list[list[int]]
```

### 13.5 数据存储

- 最高分排行榜: 本地 JSON 文件 (`save/highscores.json`)
- 当前进度: 内存中的状态快照 (关卡开始时保存)
- 设置: 本地 JSON 文件 (`save/settings.json`，音量、按键映射)

### 13.6 音频

- BGM: `pygame.mixer.music.load()` 加载 .ogg 格式音乐，`pygame.mixer.music.play(-1)` 循环播放
- SFX: `pygame.mixer.Sound()` 加载音效文件，通过 `Sound.play()` 播放
- 音频格式: .ogg (推荐) / .wav
- 预设音轨数: `pygame.mixer.set_num_channels(16)`

### 13.7 打包发布

- 使用 PyInstaller 打包为单一 .exe:
  ```
  pyinstaller --onefile --windowed --name "SuperMario" --icon=assets/icon.ico main.py
  ```
- 或使用 Nuitka 编译为原生 .exe (性能更优):
  ```
  python -m nuitka --standalone --onefile --windows-disable-console main.py
  ```
- 资源文件 (图片/音频/关卡数据) 打包进 .exe 或同目录 `assets/` 文件夹
- 最终交付: `SuperMario.exe` + `assets/` 文件夹，或单一 .exe

### 13.8 性能目标

| 指标     | 目标值        |
| -------- | ------------- |
| 帧率     | 稳定 60 FPS   |
| 启动时间 | < 3 秒        |
| 关卡切换 | < 1 秒        |
| 内存占用 | < 150 MB      |
| 输入延迟 | < 16ms (1 帧) |
| .exe 体积 | < 50 MB      |

## 14. 非功能性需求

### 14.1 兼容性

- 操作系统: Windows 10 (21H2+) / Windows 11
- 架构: x64 (主要)，兼容 x86
- 运行环境: 无需安装 Python，.exe 独立运行
- 显卡: 支持 DirectX / OpenGL 基础渲染即可

### 14.2 可维护性

- 模块化架构: 渲染器、物理引擎、输入系统、音效系统、关卡管理器相互解耦
- 关卡数据与代码分离: 关卡设计通过外部 JSON 文件配置
- 游戏循环: 基于 Pygame 标准事件循环 (`pygame.event.get()`)

## 附录 B: 核心游戏循环 (Pygame)

```python
import pygame

def main():
    pygame.init()
    screen = pygame.display.set_mode((768, 720), pygame.SCALED)
    clock = pygame.time.Clock()
    game_state = GameState()

    while True:
        # 事件处理
        for event in pygame.event.get():
            if event.type == pygame.QUIT:
                pygame.quit()
                return
            handle_event(event, game_state)

        # 逻辑更新
        if game_state.status == "playing":
            update_physics(game_state)
            update_entities(game_state)
            check_collisions(game_state)
            update_camera(game_state)
            update_timer(game_state)

        # 渲染
        render(screen, game_state)
        pygame.display.flip()
        clock.tick(60)
```
